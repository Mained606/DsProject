# Destiny Savior (DS Project) 🐉

판타지 세계관의 3인칭 액션 RPG로, 플레이어가 드래곤(펫)과 함께 성장하며 모험을 떠나는 게임입니다.

## 📌 프로젝트 개요

- **개발 기간**: 
    - 1차 프로토타입: 2025.01.06 ~ 2025.02.10 (약 1개월)
    - 2차 본 프로젝트 마무리: 2025.03.04 ~ 2025.04.23 (약 2개월)
- **개발 인원**: 6명
- **개발 환경**: 
  - Unity 2022.3 (URP)
  - Visual Studio 2022
  - Git/GitHub

## 🛠 주요 기술 스택

- **Unity Core**
  - Universal Render Pipeline (URP)
  - Input System
  - Terrain System
- **데이터 관리**
  - JSON Serialization
  - PlayerPrefs
- **디자인 패턴**
  - 상태 패턴 (게임 상태 관리)
  - 옵저버 패턴 (이벤트 시스템)
  - 싱글톤 패턴 (매니저 클래스)
  - JSON 직렬화 (세이브 시스템)
## 🎮 주요 기능

### 1. 게임 상태 관리 시스템
- 씬별 상태 관리
- 매니저 초기화 제어
- 이벤트 기반 상태 전파

### 2. 전투 시스템
- 물리/마법 데미지 계산
- 원소 속성 시스템
- 버프/디버프 관리

### 3. 세이브 시스템
- JSON 기반 데이터 직렬화
- 자동 저장 기능
- 세이브 포인트 시스템

### 4. 캐릭터 시스템
- 스탯 관리
- 스킬 시스템
- 장비 강화

### 5. 펫 시스템
- AI 로직
- 진화 시스템
- 전투 시스템

## 📂 프로젝트 구조

```
Assets/
├── DsProject/
│   ├── Scripts/
│   │   ├── Core/       # 핵심 시스템
│   │   ├── Managers/   # 게임 매니저
│   │   ├── System/     # 기능별 시스템
│   │   └── UI/         # UI 관련
│   ├── Scenes/         # 게임 씬
│   ├── Prefabs/        # 프리팹
│   └── Resources/      # 리소스 파일
```

## 🔍 주요 구현 내용

### 게임 상태 관리
- 상태 패턴을 활용한 게임 상태 머신 구현
- 메인 메뉴, 전투, 탐험, 인벤토리 등 다양한 게임 상태 관리
- 씬 전환 및 상태 변경에 따른 시스템 동작 제어

### 매니저 시스템템
- BaseManager 클래스를 통한 일관된 매니저 구조 설계
- 매니저 간 의존성 관리 및 초기화 순서 제어
- 이벤트 기반 시스템으로 컴포넌트 간 결합도 최소화

### 전투/스킬 시스템
- 물리/마법 데미지 계산 로직
- 원소 속성 상성 시스템
- 버프/디버프 효과 관리
- 전투 / 탐험 상태 전환 관리

### 데이터 관리
- JSON 기반 세이브/로드
- 게임 상태, 플레이어 정보, 인벤토리 등 저장/로드
- 세이브 포인트 시스템 구현
- 자동 저장 기능 구현

### 입력 관리 시스템
- 게임 상태에 따른 입력 제어
- UI 상호작용 및 게임플레이 입력 분리
- 커스터마이즈 가능한 키 바인딩 시스템

## 🎯 성과
- 중간 규모 프로젝트의 구조 설계 및 아키텍처 패턴 적용 경험
- 상태 패턴과 옵저버 패턴을 활용한 확장 가능한 시스템 설계
- 효율적인 매니저 시스템 구현을 통한 게임 로직 관리
- JSON 직렬화를 활용한 데이터 저장/로드 시스템 구현
